Firebase

Introduction

  • Firebase is a cross platform set of tools and core infrastructure to help developers focus on core features instead of building infrastructure.
  • Currently, it supports web apps (using JavaScript or Node.js), mobile apps (both iOS and Android) as well as C++ and Unity applications.
  • Firebase can help developers add functionality such as secure authentication, user registration, hosting, analytics and push notification functionality with just a few lines of code.
  • This would have taken developers tens of thousands of lines of code to do otherwise.

Services

  • Firebase services come bundled in one powerful SDK that contains all the Firebase tools.
  • Firebase categorizes its services into two categories:
    • – Develop: These services focus on the development of applications and help with ensuring app quality.
    • – Grow: These services focus on analytics, acquiring and re-engaging users.
  • The following products are offered by Firebase under their Develop group of services:
    • Authentication
    • Realtime Database
    • Hosting
    • Cloud Storage
    • Crashlytics (mobile only)

Develop: Realtime Database

  • It is a cloud hosted NoSQL database.
  • Does not require developers to setup a database server and maintain it.
  • Takes care of synchronizing data across multiple clients in real time.
  • Uses JSON so send data to clients.
  • Stores data locally on the client side so that the application continues to work when offline. The data is synced back once the Internet connection is available again.

Develop: Cloud Storage

  • This can be used for storage of media as well as other files.
  • Backed by Google Cloud Storage infrastructure.
  • Uses a global network of servers (CDN) to ensure that files are available at servers near the user while also ensuring redundancy for backup.
  • The APIs take care of file upload and retrieval, while automatically supporting devices on poor Internet connections and taking care of network interruptions.
